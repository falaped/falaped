# MRD — FALAPED (Marketing Requirements Document)

## Objetivo de marketing

Definir estrategia de posicionamento, alvo comercial e diretrizes de oferta para crescimento do FALAPED no Brasil como SaaS B2B2C para pediatras e clinicas pediatricas.

## Pesquisa de mercado (Brasil)

## Sinais de demanda


| Sinal                                                            | Leitura para o FALAPED                                                 |
| ---------------------------------------------------------------- | ---------------------------------------------------------------------- |
| Pediatria e uma das maiores especialidades medicas no Brasil     | Mercado amplo de profissionais para captacao vertical                  |
| Telemedicina e saude digital seguem em crescimento               | Maior abertura para fluxo clinico digital                              |
| WhatsApp e canal dominante de comunicacao no contexto brasileiro | Tese WhatsApp-first reduz friccao de adocao                            |
| Concorrentes fortes focam gestao ampla de clinica                | Espaco para produto especializado em jornada conversacional pediatrica |

## Personas de mercado

## Persona A — Pediatra solo digital


| Campo             | Descricao                                                |
| ----------------- | -------------------------------------------------------- |
| Perfil            | Pediatra com atendimento recorrente por WhatsApp         |
| Objetivo          | Reduzir tempo administrativo e manter qualidade          |
| Dor principal     | Transformar conversa em documento medico de forma rapida |
| Objeção de compra | Medo de trocar ferramenta sem ganho real                 |
| Mensagem-chave    | "Feche casos com documentacao completa em menos tempo"   |


## Persona B — Dono(a) de clinica pediatrica pequena


| Campo             | Descricao                                                      |
| ----------------- | -------------------------------------------------------------- |
| Perfil            | Clinica com 2 a 10 pediatras                                   |
| Objetivo          | Padronizar qualidade e escalar operacao                        |

| Objeção de compra | Duvida sobre curva de implementacao                            |

| Campo          | Descricao                                                           |
| -------------- | ------------------------------------------------------------------- |

| Objetivo       | Menos pendencias e mais previsibilidade                             |
| Dor principal  | Falta de visao centralizada do status por caso/documento            |

## TAM/SAM/SOM (Brasil)

## Metodologia adotada

- **Top-down:** base em tamanho de mercado de saude digital/telemedicina no Brasil + recorte pediatria.
- **Bottom-up (sanity check):** numero de pediatras alvo x ticket anual medio.

## Premissas de calculo (iniciais, para planejamento)


| Premissa                                                                  | Valor de referencia              |
| ------------------------------------------------------------------------- | -------------------------------- |
| Mercado telemedicina BR 2025 (IMARC)                                      | US$ 2,4 bi                       |
| Total de pediatras no Brasil (CFM/SBP em comunicados com base Demografia) | ~48 mil                          |
| Faixa de preco viavel do FALAPED                                          | R$ 149 a R$ 249 por medico/mes   |
| Ticket anual medio adotado para conta                                     | R$ 2.400 por medico/ano (aprox.) |


## Estimativa Bottom-up (SaaS pediatra)

`TAM (R$) = pediatras enderecaveis x ticket anual medio`

Cenario de referencia:

- Base enderecavel bruta: 48.000 pediatras.
- Ticket anual medio: R$ 2.400.
- **TAM estimado (software vertical pediatrico): R$ 115,2 milhoes/ano**.

## Recortes operacionais


| Camada         | Definicao para FALAPED                                                   | Estimativa inicial |
| -------------- | ------------------------------------------------------------------------ | ------------------ |
| TAM            | Mercado total de pediatras no BR com potencial de software especializado | ~R$ 115M/ano       |
| SAM            | Pediatras em perfil digital-first (WhatsApp ativo) nas principais pracas | 30% a 50% do TAM   |
| SOM (24 meses) | Fatia capturavel por canal e capacidade comercial inicial                | 1% a 3% do SAM     |


Observacao: valores sao de planejamento e devem ser recalibrados com dados reais de conversao, churn e ARPA.

## Posicionamento e mensagem

## Posicionamento recomendado

**"Plataforma de produtividade clinica para pediatras que atendem por WhatsApp."**

## Pilares de mensagem


| Pilar          | Prova no produto                                 |
| -------------- | ------------------------------------------------ |
| Velocidade     | Geracao de relatorio por IA e templates          |
| Organizacao    | Caso, paciente e documentos no mesmo fluxo       |
| Confianca      | Ownership por profile e downloads por signed URL |
| Especializacao | Linguagem e jornada centradas em pediatria       |


## Estrategia de pricing recomendada (BRL)

## Modelo

- Cobranca por medico/mes (padrao de mercado).
- Desconto para ciclo semestral/anual.
- Plano por volume para clinicas.

## Tabela sugerida


| Plano        | Preco sugerido    | Perfil alvo           | Limites sugeridos                |
| ------------ | ----------------- | --------------------- | -------------------------------- |
| Essencial    | R$ 149/medico/mes | Pediatra autonomo     | Fluxo principal completo         |
| Profissional | R$ 229/medico/mes | Maior volume de casos | Recursos avancados e priorizacao |
| Clinica      | Sob proposta      | 5+ medicos            | Condicoes de volume e onboarding |


## Logica de entrada comercial


| Estrategia                | Objetivo                               |
| ------------------------- | -------------------------------------- |
| Trial guiado (7-14 dias)  | Demonstrar ganho de tempo em caso real |
| Oferta anual com desconto | Melhorar caixa e reduzir churn inicial |
| Prova de ROI por caso     | Aumentar conversao para paid           |

## KPIs de marketing e crescimento


| KPI                     | Formula                          | Meta inicial                              |
| ----------------------- | -------------------------------- | ----------------------------------------- |
| CPL qualificado         | `investimento_midia / leads_ICP` | Reducao progressiva apos 60 dias          |
| Conversao trial -> paid | `contas_paid / trials`           | >= 20%                                    |
| CAC payback             | `CAC / margem_mensal_media`      | <= 6 meses                                |
| ARPA                    | `MRR / contas_pagas`             | Crescente por adocao de planos superiores |
| Churn de receita        | `MRR_cancelado / MRR_inicio_mes` | <= 5% ao mes                              |


## Riscos comerciais


| Risco                                           | Impacto    | Mitigacao                                             |
| ----------------------------------------------- | ---------- | ----------------------------------------------------- |
| Percepcao de sobreposicao com ERPs generalistas | Medio/Alto | Comunicar nicho: produtividade em caso conversacional |
| Sensibilidade a preco no segmento autonomo      | Medio      | Plano de entrada + prova de valor rapida              |
| Dependencia de canal unico de aquisicao         | Medio      | Diversificar: conteudo, indicacao, parcerias locais   |
| Falta de prova social inicial                   | Medio      | Coletar depoimentos e dados de ganho de tempo         |
