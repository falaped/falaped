<objective>
Configure GitHub branch protection on `main` so merges are blocked unless the CI pipeline (from Plan 02) is green (D-10). This is a GitHub-side configuration step that requires authenticated access to the `github.com/falaped/falaped` repo; it depends on the CI workflow existing (Plan 02) and on at least one CI run having registered the status check name.

Purpose: Branch protection is what turns CI from advisory into enforcing — TEST-03's "fails the build on push" guarantee only holds for `main` if the green check is required to merge. Without it, a red pipeline can still be merged.

Output: a branch-protection rule on `main` requiring the CI status check, configured via `gh` CLI (preferred) or the GitHub UI.
</objective>

<artifacts_this_phase_produces>
This plan produces no repository files. It creates a GitHub-side configuration object:

- Branch protection rule on `main` in `github.com/falaped/falaped` requiring the CI status check to pass before merge
</artifacts_this_phase_produces>
