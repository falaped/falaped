# Phase 1 — UI Design Contract

> Visual and interaction contract for the consultation experience surfaces: pediatric age badge, draggable consultation timer, and the gestational-age form field. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope note: CONS-04 (PDF print fix) is output/backend correctness, NOT a UI surface — it is excluded from this contract. The printed-document age intent (D-05) lands in Phase 3 and is out of scope here.
>
> This contract EXTENDS the established FALAPED design system (shadcn `radix-nova`, oklch token theme in `app/globals.css`, lucide icons, `pediatric-dashboard-design` skill). It does NOT introduce a parallel system. All values below are already present in the project; this document makes them prescriptive for the three new surfaces.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized — `components.json` present) |
| Preset | `radix-nova` style, baseColor `neutral`, cssVariables true, no prefix |

**Established primitives to reuse (do NOT rebuild):**

- `Field` / `FieldLabel` / `FieldContent` / `FieldError` / `FieldGroup` / `FieldSet` / `FieldLegend` (`components/ui/field.tsx`) — the gestational-age input MUST use these, matching `PatientFormBirthDateField`.

**Layout anchors (where these surfaces mount):**

- Gestational-age field → `components/dashboard/patients/patient-form/patient-form-personal-section.tsx` (new field in the "Identificação e contato" section, next to the antropometria grid).

## Spacing Scale

Declared values (multiples of 4; map to existing Tailwind utilities — never raw px in components):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon-to-text gap inside badges and chips |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing; badge internal padding; chip gaps |
| md | 16px | `gap-4` / `p-4` | Default element/field spacing; form grid gaps; widget body padding |
| lg | 24px | `gap-6` / `p-6` | Section padding (hero, cards); form section vertical rhythm |
| xl | 32px | `gap-8` | Layout gaps between major blocks |
| 2xl | 48px | `gap-12` | Major section breaks (not expected on these surfaces) |
| 3xl | 64px | `gap-16` | Page-level spacing (not expected on these surfaces) |

**Exceptions for this phase:**
- Floating timer widget drag-handle / tap targets MUST be at least 44px in the smallest dimension (touch-target minimum) even though 44 is not on the 4-scale — applied via `min-h-11 min-w-11` (44px) on the handle and pause/resume control. This is the one allowed exception.
- Badge internal gap uses `gap-1.5` (6px) ONLY to match the existing `PatientDetailHero` and `CaseDetailHeader` badge pattern (`gap-1.5`) for visual consistency with shipped code; do not introduce 6px anywhere new.

## Typography

All sizes are existing Tailwind classes already used across the dashboard. Exactly 4 roles, 2 weights (normal 400 + semibold 600 — matching the `pediatric-dashboard-design` skill: `font-semibold` headings, `font-normal` badges/body).

| Role | Size | Weight | Line Height | Tailwind | Used by |
|------|------|--------|-------------|----------|---------|
| Display / page title | 24px (sm:30px) | 600 semibold | 1.2 (`tracking-tight`) | `text-2xl font-semibold tracking-tight` (`sm:text-3xl` on hero) | Patient name (hero), case title — NOT changed this phase |
| Body / badge text | 14px | 400 normal | 1.5 | `text-sm font-normal` | Age badge text, responsible/contact lines, helper text container |
| Label | 14px | 400→500 | 1.4 | `text-sm` (FieldLabel) | Gestational-age field label |
| Caption / hint | 12px | 400 normal | 1.4 | `text-xs text-muted-foreground` | Field hint ("Idade gestacional ao nascer"), corrected-age sub-label, timer "pausado" tag |

**Timer numeric readout:** use `tabular-nums` + `font-mono text-sm` (matches the birth-date input's `font-mono tabular-nums`) so the digits do not jitter as seconds tick. Elapsed time displayed at `text-base` (16px) `font-semibold tabular-nums` inside the widget for at-a-glance legibility.

## Color

The theme is oklch token-based (`app/globals.css`). Components MUST use semantic tokens — never hex/rgb/oklch literals (project rule from `creative-director-falaped` and `pediatric-dashboard-design`).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` (oklch 0.985 light / 0.145 dark) | App/page background, behind the floating widget |
| Secondary (30%) | `bg-card` / `bg-muted` / `bg-secondary` | Hero card surface, badge `secondary` fill, timer widget surface (`bg-card`), form section |
| Accent (10%) | `bg-primary` / `text-primary` (soft blue, oklch 0.76 0.09 255) | See reserved-for list below |
| Destructive | `text-destructive` / `bg-destructive` (oklch 0.58 0.22 27) | Invalid/future birth-date error only |

**Accent (primary, soft blue) reserved for — explicit list (never "all interactive elements"):**
1. The "Caso ativo" running-status indicator already in the case header (existing) — the timer widget's **running** state border/dot reuses this exact treatment (`bg-primary` pinging dot, matching `StatusBadge`).
2. The single primary CTA when birth date is missing: the **"Completar cadastro"** button (D-09 empty state) — `Button` default variant (`bg-primary text-primary-foreground`).
3. Avatar initials fallback (existing `bg-primary/10 text-primary`) — unchanged.

Everything else (age badge, gestational field, timer paused/ended states, drag handle) uses **neutral** tokens (`secondary` / `muted` / `border` / `foreground` / `muted-foreground`). The age badge is informational, NOT an action — it MUST stay `Badge variant="secondary"` (matching the existing DOB badge in `patient-detail-hero.tsx`), never accent.

**Destructive reserved for:** the inline error/warning state on an invalid or future birth date (D-12) only — `text-destructive text-sm` message below/beside the badge, and a destructive-toned `Badge variant="outline"` with `text-destructive` for the inline chip. No destructive actions exist in this phase (no delete/irreversible operations on these three surfaces); pause/resume and close-case are non-destructive.

## Copywriting Contract

All copy PT-BR, professional, domain terms per `audience-context` (paciente, responsável, idade corrigida). Verbatim strings below are the contract — the executor uses these exact words.

| Element | Copy |
|---------|------|
| Primary CTA (missing DOB, D-09) | **"Completar cadastro"** (button) with preceding text "Data de nascimento não informada." |
| Empty state heading (age, no DOB) | **"Idade indisponível"** |
| Empty state body (age, no DOB) | **"Informe a data de nascimento para calcular a idade."** + CTA "Completar cadastro" (links to patient edit / focuses the birth-date field) |
| Error state (invalid DOB, D-12) | **"Data de nascimento inválida — verifique o cadastro."** (`text-destructive`) |
| Error state (future DOB, D-12) | **"Data de nascimento no futuro — verifique o cadastro."** (`text-destructive`) |
| Destructive confirmation | not applicable — no destructive actions on these surfaces this phase |

**Age badge copy rules (D-06/D-07/D-08/D-11):**
- Profile (hero): full text by band — `"12/03/2025 · 3 meses e 12 dias"`. DOB formatted via existing `formatDate`, then ` · `, then the by-extenso age from the new `lib/compute-pediatric-age.ts` engine.
- Band wording (PT-BR, by extenso, singular/plural agreement REQUIRED):
  - 0–28 days → `"5 dias"` / `"1 dia"`
  - 1–~12 weeks → `"6 semanas"` / `"1 semana"`
  - ~3–24 months → `"3 meses e 12 dias"` / `"1 mês e 1 dia"` (omit the days clause when 0 days → `"4 meses"`)
  - ≥24 months → `"2 anos e 4 meses"` / `"1 ano e 1 mês"` (omit months clause when 0 → `"3 anos"`)
- Corrected age (D-10, preterm): append a second, distinctly labelled chip/line — **"idade corrigida"** prefix, e.g. `"idade corrigida: 2 meses e 4 dias"`. The chronological age stays primary; corrected age is the secondary line. When the child is full-term or gestational age is absent, show ONLY chronological (no "corrigida" label). Stop showing corrected age after ~24 months corrected.
- Chronological label when corrected is also shown: prefix the primary with **"idade cronológica"** so the two are unambiguous (only when both are present; when corrected is absent, the primary age needs no prefix).

**Case-header age (abbreviated — Claude's discretion, D-08):** in the tight case header next to the timer, abbreviate to conserve width: `"3m 12d"`, `"6 sem"`, `"5 d"`, `"2a 4m"`. Wrap in `Badge variant="secondary"` with a `Tooltip` exposing the full by-extenso text and DOB. Corrected age in the header: append `" (corr.)"` to the abbreviated value with the full "idade corrigida" text in the tooltip.

**Timer copy (D-02/D-03):**
- Running: no label needed; show elapsed `mm:ss` (or `h:mm:ss` past 1h) with the pinging primary dot.
- Paused: chip **"Pausado"** (`Badge variant="outline" text-muted-foreground`), elapsed frozen.
- Ended (case closed): chip **"Encerrado"**, elapsed shows final duration, controls hidden.
- Controls: pause button label/`aria-label` **"Pausar consulta"**; resume **"Retomar consulta"**. Icons: `PauseIcon` / `PlayIcon` (lucide).

**Gestational-age field copy (D-10):**
- Label: **"Idade gestacional ao nascer"**
- Hint (`text-xs text-muted-foreground`): **"Em semanas. Usada para calcular a idade corrigida de prematuros."**
- Optional/not-required: field is optional; no error when empty.
- Validation error (out of plausible range): **"Informe um valor entre 20 e 42 semanas."** (Zod, mapped via `lib/zod-error-message.ts`)
- Placeholder: **"Ex.: 34"**

## Component Contracts (phase surfaces)

### 1. Pediatric Age Badge

| Aspect | Contract |
|--------|----------|
| Primitive | `Badge variant="secondary"` + lucide `CalendarIcon` (`h-3.5 w-3.5 shrink-0 opacity-70`), `gap-1.5`, `font-normal` — identical to the existing DOB badge in `patient-detail-hero.tsx` |
| Profile context | Full by-extenso text, `wrap-break-word`, DOB · age in one badge; corrected age (if any) as a second adjacent badge/line labelled "idade corrigida" |
| Case-header context | Abbreviated text, single `Badge`, `Tooltip` with full text; sits in the `flex flex-wrap items-center gap-x-4` meta row next to the timer reference |
| Lists | NOT rendered in patient/case lists (D-05) |
| Missing DOB (D-09) | Replace badge with the empty-state pattern from `pediatric-dashboard-design` (muted text + `Button` CTA "Completar cadastro"); never blank, never crash |
| Invalid/future (D-12) | Show `text-destructive` message + outline destructive chip; never treat as missing, never silently hide |
| Source of truth | `lib/compute-pediatric-age.ts` returns `{status, band, parts, corrected?}`; the badge is a pure render of that result — no inline date math in the component |

### 2. Consultation Timer Widget (floating, draggable)

| Aspect | Contract |
|--------|----------|
| Container | `Card`-toned floating panel: `bg-card border border-border rounded-lg shadow-md`, `p-3`, fixed positioning, `z-50` (above content, below modals/toasts) |
| Footprint | Compact: roughly `w-auto` min ~180px; elapsed readout `text-base font-semibold tabular-nums`; never obscures the primary content column — default anchored bottom-right with `inset` offset (`bottom-6 right-6`), clear of the case panel |
| Drag (D-04) | `@dnd-kit` `useDraggable`; whole header bar is the drag handle (`cursor-grab` / `cursor-grabbing`), `min-h-11` touch target; `GripVerticalIcon` (lucide) as the visible drag affordance at `opacity-60` |
| Position persistence | Claude's discretion → **persist `{x,y}` in `localStorage`** keyed per device on drag end; clamp to viewport on load so it never renders off-screen. Per-device only (acceptable, A5). |
| Running state | Primary pinging dot (reuse `StatusBadge` active treatment), live `setInterval(1s)` repaint, elapsed computed from persisted `started_at` (drift-free) |
| Paused state | "Pausado" outline chip, elapsed frozen, dot static `bg-muted-foreground`, primary control becomes "Retomar consulta" |
| Ended state | "Encerrado" chip, final elapsed, pause/resume controls removed; widget may auto-dismiss or show read-only |
| Controls | One toggle button (`Pausar consulta` ⇄ `Retomar consulta`) with lucide `PauseIcon`/`PlayIcon`; `Button variant="ghost" size="icon"`, `min-h-11 min-w-11`; `aria-label` always set; `focus-visible:ring-2` |
| Motion | `transition-colors` only; no decorative animation beyond the existing ping dot |
| Reduced motion | Honor `prefers-reduced-motion`: suppress the ping animation, keep a static dot |

### 3. Gestational-Age Input

| Aspect | Contract |
|--------|----------|
| Placement | Inside `PatientFormPersonalSection` "Identificação e contato" `FieldSet`, on the same row pattern as birth-date/sex (contained width `w-full md:w-[12rem]`), after the birth-date field |
| Primitive | `Field` + `FieldLabel` + `FieldContent` + `Input` + `FieldError` — identical structure to `PatientFormBirthDateField`; `inputMode="numeric"`, `font-mono text-sm tabular-nums` |
| Label / hint | "Idade gestacional ao nascer" + hint "Em semanas. Usada para calcular a idade corrigida de prematuros." (`aria-describedby` to the hint id, matching the birth-date field pattern) |
| Validation | Optional integer; Zod `safeParse` 20–42 weeks; PT-BR error via `lib/zod-error-message.ts`; error shown as `text-destructive text-sm` below the field (existing forms pattern) |
| Empty | No error, no corrected-age computed — corrected age simply unavailable (graceful, matches the nullable column) |

## Accessibility & Responsive (cross-cutting)

- All interactive controls: `focus-visible:ring-2` (existing convention); `aria-label` on icon-only buttons (pause/resume, drag handle).
- Timer drag handle is keyboard-operable via `@dnd-kit` keyboard sensor (do not ship pointer-only).
- Color is never the sole signal: running/paused/ended each have a text chip in addition to the dot color.
- Age badge meaning conveyed by text, not icon alone; tooltip text is also the accessible name in the abbreviated header.
- Responsive: on narrow widths the case-header age badge wraps within the existing `flex-wrap` meta row; the floating widget clamps inside the viewport and keeps its 44px touch targets.
- Honor `prefers-reduced-motion` for the ping dot.
