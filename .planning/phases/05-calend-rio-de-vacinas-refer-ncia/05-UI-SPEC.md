# Phase 5 — UI Design Contract

> Visual and interaction contract for the **Calendário de Vacinas (Referência)** phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope: a **read-only reference view** of three vaccine datasets — SUS/PNI and particular/SBIm shown **side by side** (VAC-01/VAC-02), plus a separate **Gestante** tab (VAC-03) — modeled as versioned data with source + vigência visible (VAC-04). Two entry points: standalone route `/dashboard/vaccines` (no patient) and from the patient profile (child's current age band highlighted). Out of scope (do NOT spec UI for): per-patient vaccination record / applied doses / pending-late diff / next-dose highlight (that is Phase 6, which *consumes* this data). No write UI in this phase — data is seed-only.

## Typography

Uses the dashboard's existing hierarchy (pediatric-dashboard-design skill: page header `text-2xl font-semibold tracking-tight` + `text-sm text-muted-foreground`). 4 roles, 2 weights (`font-normal` 400 body, `font-semibold` 600 headings/emphasis). Line-heights follow Tailwind defaults confirmed by the skill.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 (`leading-normal`) — vaccine names, dose/age text, gestante window text, table cells, descriptions |
| Label | 12px (`text-xs`) | 400 (`font-normal`) | 1.4 — age-band labels, column/table headers (`uppercase tracking-wide text-muted-foreground`), the provenance caption (`Fonte: … · vigência …`), the persistent advisory — distinct via size + letter-spacing + muted color, not by heavier weight |
| Heading | 18px (`text-lg`) | 600 (`font-semibold`) | 1.2 (`tracking-tight`) — the two column headers ("SUS/PNI", "Particular (SBIm)") and per-age-band subheadings within a column |
| Display | 24px (`text-2xl`) | 600 (`font-semibold`) | 1.2 (`tracking-tight`) — page `<h1>` ("Calendário de vacinas") per the dashboard header pattern |

Note: any age value shown numerically (e.g. "2 meses", "20 semanas") stays in Body; no `tabular-nums` display tiles in this phase (no large numeric readouts).

## Color

60/30/10 split follows the existing dashboard tokens; this is an information-dense reference view, so accent is deliberately minimal and reserved for **orientation**, not decoration.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` (`oklch(0.985 0 0)` light) | page/section surface behind the tabs and column grid |
| Secondary (30%) | `bg-card` / `bg-muted/50` (`oklch(1 0 0)` card, `oklch(0.97 0 0)` muted) | each column card (SUS, SBIm), gestante list card, provenance footer block, alternating age-band row zebra (`even:bg-muted/50`) |
| Accent (10%) | `--primary` blue | (a) active tab indicator (Criança ↔ Gestante); (b) the **current-age-band highlight** when opened from a patient — a left border + `bg-primary/10` on the matching age band in *both* columns (D-02); (c) any link/affordance back to the patient |
| Destructive | `--destructive` red | error state only (load failure). No destructive user actions exist in this phase. |

Accent reserved for (explicit — nothing else may take accent):
1. The active segment of the top tabs (`Criança (SUS × SBIm)` ↔ `Gestante`).
2. The **current-age-band emphasis** (border-l + `bg-primary/10`) when the calendar is opened with a patient's age (D-02) — applied to the same band in the SUS *and* SBIm columns so the eye reads "onde estamos" across both.
3. Navigation affordance back to / from the patient profile.

Do NOT color SUS vs SBIm with two different hues, and do NOT color-code individual vaccines. The two datasets are distinguished by **column position + header label**, not color. The persistent advisory is a **muted** info treatment (`text-muted-foreground` / `bg-muted`), NOT destructive red and NOT a saturated warning — it is standing guidance, not an error. This app has no standalone success/warning token; do not introduce green/amber.

Age-band highlight is the ONLY place `bg-primary/10` appears in this phase; keep the rest of the surface neutral so the highlight actually reads.

## Interaction & Layout Contracts

Prescriptive contracts for the phase-specific screens. Executor implements against these; ui-checker/ui-auditor validate against them.

### C1 — Page shell & entry points (D-03)
- Two entry points, one shared view component:
  1. **Standalone:** route `app/dashboard/vaccines/page.tsx`, added to the sidebar nav (mirror the `referrals` / `prescriptions` top-level route pattern). No patient context → no age highlight.
  2. **From patient profile:** opened with the child's DOB/age → renders the same view with the current age band highlighted (C4). Mirror the avulso + por-paciente pattern of Phase 4 (D-13).
- Page shell: `flex flex-col gap-6`, `max-w-6xl`, `p-6`/`p-8`; header = h1 + muted description per the dashboard header pattern. Breadcrumb consistent with sibling routes.

### C2 — Tabs: Criança vs Gestante (D-04)
- Top-level shadcn `Tabs`, two tabs: `Criança (SUS × SBIm)` (default) and `Gestante`. Active tab uses the accent indicator.
- Rationale surfaced in copy: gestante is a **separate dataset on a different axis** (weeks of gestation, not child age) — the tab boundary makes that explicit (VAC-04 criterion 3).

### C3 — Side-by-side columns: SUS × SBIm (D-01)
- Inside the "Criança" tab: **two parallel columns** — SUS/PNI left, particular/SBIm right — each its own **list grouped by age band** (NOT a vaccine×age grid, NOT a single accordion).
- Layout: two-column grid at `md+` (`grid grid-cols-1 md:grid-cols-2 gap-4`); on narrow/mobile the columns stack (SUS above SBIm) but stay clearly labeled per column header — never merge into one list.
- Each column is a card with a sticky-or-repeated column header (`SUS/PNI` / `Particular (SBIm)`). Within a column, age bands are the grouping unit; each band lists its vaccines/doses as body text.
- **Alignment for comparison:** age bands should read across at the same vertical rhythm where possible so the doctor can compare "what's due at 2 meses" in SUS vs SBIm at a glance. Where one dataset has no vaccine for a band, render an explicit empty marker (`—` or omit the band consistently) rather than silent misalignment.

### C4 — Current-age highlight (D-02 / D-11)
- Only in patient mode. Reuse the Phase 1 age engine (`lib/compute-pediatric-age.ts`) to compute which age band is current — **for positioning/highlight only**, no diff/pending logic (that is Phase 6).
- Highlight = `border-l-2 border-primary` + `bg-primary/10` on the matching band in **both** columns, plus an `Idade atual` badge. Do NOT filter to only the current band — the whole calendar stays visible (past + future); the current band is emphasized and scrolled into view.
- Non-patient (standalone) mode: no highlight, no badge.

### C5 — Gestante list (D-05, VAC-03)
- Inside the "Gestante" tab: a **single list by vaccine with the window in text** (NOT grouped by trimester — some vaccines cross trimesters). Exact rows per Copywriting table.
- Card + list (or table with columns Vacina / Janela). Body text; the window phrase (`a partir de 20 semanas`, `28–36 semanas`, `qualquer momento`) is part of the row, muted where it's a qualifier.

### C6 — Provenance & advisory (D-08 / D-09, VAC-04)
- Each dataset (SUS, SBIm, gestante) shows its own provenance caption read from schedule metadata — `Fonte: {source} {version} · vigência {effective_date}` — in the footer/header of that dataset's block, NOT a single global banner detached from the data.
- The persistent advisory `Confira sempre contra o calendário oficial atual.` is always visible (muted/info), placed with the provenance captions. Not dismissible.
- Provenance is REQUIRED on-screen for all three datasets (ui-checker will verify this against VAC-04 criterion 3).

### C7 — States coverage (checker requirement)
Every surface declares: **loading** (shadcn `Skeleton` for the columns/list, never a full-screen blocker), **populated** (the normal reference view), **error** (inline message + retry, per Copywriting), and a defensive **empty** (dataset unseeded). There is no "empty because user has no data" state — this is shared reference data. Hover/focus: `transition-colors`, `focus-visible:ring-2` on interactive elements (tabs, nav).

### Accessibility
- Tabs are the shadcn/radix `Tabs` primitive (keyboard-operable, proper `aria`); the two columns use a `<table>` or semantic list with a real header association so screen readers announce the dataset per row.
- The current-age emphasis must not rely on color alone — pair `bg-primary/10` with the `Idade atual` text badge (color + text).
- Each dataset block is a labeled region (`<section aria-labelledby>`); the persistent advisory is readable text, not an icon-only affordance.
