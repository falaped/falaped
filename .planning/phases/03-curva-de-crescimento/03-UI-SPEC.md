# Phase 3 — UI Design Contract

> Visual and interaction contract for the Curva de Crescimento (pediatric growth curve) phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope: a chart-heavy growth section living **inside the patient profile** (`app/dashboard/patients/[id]`), beside the existing `patient-clinical-overview.tsx` scalar card (which must remain untouched — D-08). Covers measurement entry + history + 4 WHO growth charts + prematurity corrected-age handling + per-measurement position readout. Out of scope (do NOT spec UI for): PDF export, in-consult recording, deviation alerts.

## Spacing Scale

Declared values (Tailwind spacing utilities only; all multiples of 4). Matches the pediatric-dashboard-design skill (`gap-6` sections, `p-4`/`p-6` cards, `space-y-*`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `mt-1`) | icon↔label gap, sub-label offset |
| sm | 8px (`gap-2`, `pb-2`) | compact tile internals, card header pad |
| md | 16px (`p-4`, `gap-4`) | card padding, form field gaps, chart tile padding |
| lg | 24px (`gap-6`, `space-y-6`) | spacing between growth section blocks (form / history / charts) and from the clinical-overview section |
| xl | 32px (`p-8`) | empty-state padding (matches existing dashed empty states) |

Exceptions:
- Recharts internal chart height is a fixed pixel value (`height={360}` per RESEARCH pattern) — this is a chart-canvas dimension, not layout spacing, and is exempt from the 4px grid.
- Recharts axis/tick/margin pixel props (chart-internal) are governed by Recharts defaults, not this scale.

## Typography

Uses the dashboard's existing hierarchy (pediatric-dashboard-design skill + `patient-clinical-overview.tsx`). 4 roles, 2 weights (`font-normal` 400 body, `font-semibold` 600 headings/values). Line-heights follow Tailwind defaults confirmed by the skill.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 (`leading-normal`) — descriptions, table cells, helper text, `text-muted-foreground` |
| Label | 12px (`text-xs`) | 500 (`font-medium`) | 1.4 — uppercase tile labels (`uppercase tracking-wide`), chart legends, axis captions, source/vintage caption |
| Heading | 18px (`text-lg`) | 600 (`font-semibold`) | 1.2 (`tracking-tight`) — the growth section `<h2>` ("Curva de crescimento"), matching the clinical-overview `<h2>` |
| Display | 30px (`text-3xl`) | 600 (`font-semibold`) | 1.1 (`tracking-tight tabular-nums`) — large numeric readouts (percentile/z-score value tile), matching the existing MetricTile value size |

Note: `tabular-nums` is REQUIRED on every numeric readout (measurements, percentile, z-score, IMC) so digits align in the history table and tiles — consistent with the existing `MetricTile`.

## Color

60/30/10 split follows the existing dashboard tokens; accent is deliberately narrow.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` (`oklch(0.985 0 0)` light) | page/section surface behind the growth section and chart canvas |
| Secondary (30%) | `bg-card` / `bg-muted/15` (`oklch(1 0 0)` card) | measurement form card, history table card, chart container card, metric tiles |
| Accent (10%) | `--primary` blue + `--chart-1..5` blue ramp | Primary CTA button; the **P50 median reference line** (emphasized `strokeWidth={2}`); the **patient scatter points** and their connector; active state of the percentil/z-score and cronológica/corrigida toggles |
| Destructive | `--destructive` red | "Remover" measurement action + its confirmation dialog; out-of-band position status (`bad`/`invalid`); inline validation errors (`text-destructive text-sm`) |

Accent reserved for (explicit — nothing else may take accent):
1. The single primary CTA per surface ("Registrar medição" / "Salvar medição").
2. The patient's own plotted points + the P50 median line on each chart (the clinically most-read line).
3. The active segment of the two chart toggles (percentil↔escore-z; cronológica↔corrigida).

Reference-band lines (P3/P15/P85/P97 and z −3..+3, excluding P50) use the muted end of the `--chart` ramp / `--muted-foreground` at `strokeWidth={1}` so the dense reference lines recede and the child's data reads first. Do NOT color each centile band a different hue — density legibility (D-12) depends on a restrained monochrome ramp with only P50 emphasized.

Position-readout status color (D-13) reuses `lib/patient-bmi-ui-status.ts` mapping verbatim: `good → border-chart-2/50 bg-chart-2/10`, `warn → border-chart-4/55 bg-chart-4/10`, `bad`/`invalid → border-destructive/50 bg-destructive/10`. Do NOT introduce a new green/amber/red — the app has no standalone success/warning token; `chart-2`/`chart-4` ARE the established stand-ins.

## Interaction & Layout Contracts

Prescriptive contracts for the phase-specific screens. Executor implements against these; ui-checker/ui-auditor validate against them.

### C1 — Growth section placement (D-08 / D-09)
- New `growth-section.tsx` renders **below** `PatientClinicalOverview` in the patient detail page, separated by `space-y-6` / `gap-6`. It is its own `<section aria-labelledby>` with the h2 pattern.
- The existing scalar antropometria + IMC card is **not modified or moved** (D-08). The two coexist visibly: one is "reference value" (card), the other is "history + curve".

### C2 — Measurement entry form (GROWTH-01 / D-10 / D-11)
- Reuses the patient-form field pattern: `Field` + `FieldLabel` + `Input` (`components/ui/field.tsx`), react-hook-form + Zod, labels above input, discreet placeholders.
- Fields: **Data da medição** (masked `dd/mm/aaaa` text input via `lib/brazilian-date-form.ts`, NOT a bare `type="date"`; past dates allowed, future dates rejected — D-10), **Peso** (kg), **Comprimento/estatura** (cm), **Perímetro cefálico** (cm). The three measurements are each optional but at least one is required (Zod + table CHECK).
- Short numeric fields use contained width in a compact grid (`grid` asymmetric columns at `md+`), consistent with the patient-form antropometria row.
- **Derived IMC** shows read-only inside the form/summary only when peso AND estatura are both present (D-11) with the derived hint copy; never an input.
- Primary button `Salvar medição`; `Salvando...` during submit; success = discreet toast (`sonner`) + form reset/close; inline field errors `text-destructive text-sm`.
- Entry surface may be an inline card or a modal/sheet — either is acceptable; the CTA that opens it is `Registrar medição`.

### C3 — History list (GROWTH-01 / D-14)
- Table-in-card pattern (pediatric-dashboard-design Data display): header `text-sm font-medium text-muted-foreground`, cells `px-4 py-3`, numeric columns `tabular-nums`, optional `even:bg-muted/50`.
- Columns: Data (formatted PT-BR), Peso, Estatura, PC, IMC (derived, `—` when N/A), and per-row actions `Editar` / `Remover`.
- Empty rows render `—`, never blank.
- `Remover` triggers the destructive AlertDialog (C6-style) — mirror Phase 2's remove-photo AlertDialog pattern.
- Sorted by `measured_on` (chronological); most-recent orientation at doctor's discretion but consistent.

### C4 — The 4 charts: switch & layout (GROWTH-02 / D-02)
- 4 indicators: **peso/idade, estatura/idade, IMC/idade, PC/idade** — switch via shadcn `Tabs` (one indicator visible at a time) to keep each chart legible; do NOT tile 4 dense charts simultaneously on the default view.
- One `growth-chart.tsx` (indicator-agnostic `"use client"` Recharts `ComposedChart`) reused per tab; `ResponsiveContainer width="100%" height={360}`.
- x-axis = idade (meses), numeric, domain `[ageMin, ageMax]` from the reference file (respect each curve's ceiling — D-03). y-axis = the indicator value with unit.
- Reference provenance caption (Fonte/faixa) rendered adjacent to every chart (D-03 mandatory).
- IMC and PC tabs must gracefully handle "no data for this indicator" with the chart-empty copy.

### C5 — Percentil ↔ escore-z toggle (D-12)
- A single 2-option toggle (shadcn `ToggleGroup` or segmented control) with active segment in accent. Default = **Percentil** (P3/P15/P50/P85/P97) — the Caderneta da Criança mental model.
- `Escore-z` shows −3..+3 DP lines. **Same LMS dataset** feeds both — the toggle only re-evaluates which z-values the band lines are drawn at (no data reload flicker).
- P50 (or z=0) is the emphasized accent line; outer bands are muted `strokeWidth={1}`.

### C6 — Prematurity corrected-age toggle (GROWTH-03 / D-04 / D-06)
- When `gestational_age_weeks < 37`, corrected age is applied **automatically** and the default view emphasizes the **corrected** point; a manual `Idade cronológica ↔ Idade corrigida` toggle switches which basis is primary.
- **Two points without clutter (D-04):** for a given measurement plot both x-positions (chronological + corrected) as a connected pair — a faint connector line + two markers, the active-basis marker emphasized (accent), the other de-emphasized (muted). The toggle is the declutter mechanism; do NOT draw both pairs for every historical point when it clutters — respect the active basis.
- Show the corrected-age note copy when the toggle/auto-correction is in effect. Correction ceiling = 36 months (D-05); beyond that only chronological.
- For non-preterm patients, the toggle is hidden entirely (single chronological point).

### C7 — Per-measurement position readout (D-13)
- For the selected/most-recent measurement, show a status tile reusing the MetricTile visual + the BMI-card color language: percentil + escore-z value (`text-3xl tabular-nums`) and a classification label (e.g. `Peso no P75`, `IMC: eutrófico`).
- Status color: WHO z-cutoff classification mapped onto the existing `good/warn/bad` → `chart-2/chart-4/destructive` tokens (reuse `patient-bmi-ui-status`'s color map only; the WHO z-score is the source of truth for the curve, not the card's heuristic bands).
- Out-of-range: label `> P97` / `< P3` with the exact z-score still shown.

### C8 — States coverage (checker requirement)
Every screen declares: **loading** (shadcn `Skeleton` for chart + table, never a full-screen blocker), **empty** (per Copywriting table — distinguish "no measurements", "no sex", "no data for this indicator"), **error** (inline message + retry), **populated**. Hover/focus: `transition-colors`, `focus-visible:ring-2` (skill Loading/feedback rules).

### Accessibility
- Chart is decorative-plus-data: provide an accessible text alternative (the history table + position readout convey the same data textually; the chart is not the sole information channel).
- All form inputs have associated `FieldLabel`; toggles are keyboard-operable (radix/shadcn primitives); `aria-labelledby` on the section.
