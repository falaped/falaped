## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Server Action | `id` de medição (potencialmente alheio) e updates cruzam aqui |
| Server Action → módulo → DB | update/delete em `patient_measurements`; ownership reforçada server-side |
| authenticated user → outro tenant | médico A não pode editar/apagar medição do médico B |
